git入门与进阶(七)

2023/11/1 版本控制系统 git 分布式管理系统 远程仓库

Echo已经学会了在本地创建Git仓库，随即又有新的需求想要动手实现了。

他想在Github创建一个Git仓库，让本地和github的这两个仓库进行远程同步。如此一来，Github可以作为本地的备份，还能让其他人远程协作（你也可以不带电脑在外地方便地更新blog了）。真是一举多得呀！Echo这么想着，已经摩拳擦掌、跃跃欲试了。

首先打开github主页，点击右上角的“+”，create new…–>New repository。

创建完成后，github贴心地给出了很多tips。

然后我们这里选择第二个方式。

在本地仓库打开git bash。

git remote add origin git@github.com:XXXXXXXXXXXXXXXXXX 
git branch -M main
git push -u origin main

xxxxxxxxxxx这里填的是你自己的仓库地址

origin是远程库的叫法。你可以修改它但其实没有必要。

git branch -M main 命令用于将当前分支重命名为 main。在最新的 Git 版本中，主分支的名称已从 master 改为 main。

最后，git push -u origin main 命令将本地代码库的 main 分支推送到远程仓库的 main 分支，并使用 -u 参数设置该远程分支为默认上游分支，以后的 git push 命令就可以简化为 git push。

请确保在执行这些命令之前已经初始化了本地代码库（通过 git init 命令）并添加了要提交的文件（通过 git add <filename> 命令），然后再依次执行以上这些命令。

到git push这一步结束，echo发现，自己的github已经出现了一个和本地仓库一模一样的远程库。终于大功告成！

从现在起，只要本地作了提交，就可以通过命令：

$ git push origin master

把本地master分支的最新修改推送至GitHub，现在，你就拥有了真正的分布式版本库！
